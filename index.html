üß≠ Project Methodology (Hybrid: Incremental + Iterative)

Requirements: Deliverables are well-defined (certificate, applied project, dashboard, portfolio), but dataset choice, dashboard design, and competency documentation still require iteration and feedback.

Delivery Cadence: To reduce risk, the project is divided into three major releases instead of one final delivery.

  • Release 1 (Week 5): Certification progress + competency framework setup
  • Release 2 (Week 10): Applied project draft + dashboard prototype
  • Release 3 (Week 14): Completed certificate + final deliverables + competency documentation

Approach:

  • Predictive: Applied to certification since course modules are fixed and structured.
  • Iterative: Used for applied project and dashboard to refine through feedback cycles.
  • Incremental: Portfolio and competency documentation built up progressively, finalized at semester end.

Why Hybrid? This approach balances predictability with adaptability‚Äîideal for a semester project that combines fixed requirements (certification) with evolving creative elements (competency demonstration).
