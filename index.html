📋 1. Project Goal & Description

Project Goal: Identify, analyze, and close the skill gaps necessary to secure a career in the information technology field, focusing on roles aligned to my background and interests: Business Operations Analyst (Starlink), Business Analyst (Starlink), and IT Services Technician.

This project uses real job postings to define role-specific competencies, performs a personal GAP analysis comparing current capabilities to requirements, and executes a targeted development plan. The plan culminates in tangible, career-aligned outputs: a skill development evidence set, updated resume and LinkedIn, and role-specific application materials.

Ultimate Purpose: Align my education and SpaceX experience with employer expectations to obtain a full-time role in Business Analysis, Business Operations, or IT Services.

🎯 2. Vision, Mission & Strategy Alignment

Personal Mission Connection:

Apply technology and data to improve decisions, efficiency, and user experience in high-stakes environments (manufacturing, customer operations, engineering support).

Vision Alignment (3–5 years):

Progress into a Business/IT role that blends analytics, operations, and service excellence—building toward leadership in data-driven decision-making and IT service delivery.

Strategic Goal Support (Semester):

  • Translate three target roles into a concrete competency map (skills, tools, responsibilities).
  • Close priority gaps (SQL, BI dashboards, Python, ITSM fundamentals) with measurable outputs.
  • Package outcomes into role-ready materials to support successful applications and interviews.

💼 3. Business Case & Justification

Professional Value:

  • Targets in-demand competencies drawn directly from enterprise job descriptions.
  • Produces verifiable artifacts (dashboards, SQL queries, Python notebooks, ITSM documentation) to evidence skills.
  • Improves ability to communicate analytical insights and service impacts to leadership.

Personal Value:

  • Leverages current SpaceX experience (inventory/IT hardware workflows) into BA/IT Services career paths.
  • Provides a structured path from coursework to employment with clear milestones.

Learning Outcomes:

  • Intermediate SQL (joins, views, aggregations) and data modeling for analytics use cases.
  • Interactive BI dashboards (Tableau or Power BI)
  • Python for data automation (ETL basics)
  • IT service fundamentals (ITIL/ITSM concepts, asset lifecycle and ticket workflows).

📏 4. High-Level Scope

✅ In Scope for Semester Project

  • Analyze three target roles (Business Ops Analyst, Business Analyst, IT Services Technician).
  • Create a role-by-skill matrix and perform a personal GAP analysis.
  • Design and execute a skills development plan (SQL, BI dashboard, Python automation, ITSM primer).
  • Produce tangible artifacts: dashboard, SQL workbook, Python mini-automation, ITSM SOP.
  • Update resume & LinkedIn; prepare role-specific cover letter templates.

🗺️ Roadmap Items (≤6 Months)

  • Attempt one entry-level certification (CompTIA A+ or ITIL 4 Foundation or DA-100 equivalent).
  • Add a second BI dashboard (Power BI if first is Tableau, or vice versa).
  • Expand Python automation into a small ETL pipeline to a hosted dashboard.

❌ Out of Scope

  • Guaranteeing employment outcomes (applications may occur, but hiring is outside project control).
  • Building enterprise-grade data platforms beyond the scope of a semester.
  • Pursuing multiple advanced certifications during the semester.

🧭 Project Methodology (Hybrid: Incremental + Iterative)

Requirements: Role analysis and deliverables are well-defined, but artifacts (dashboard, SQL set, Python automation) benefit from iteration and feedback.

Delivery Cadence: Three releases to reduce risk and incorporate feedback.

  • Release 1 (Week 3–5): Role research matrix + GAP analysis draft
  • Release 2 (Week 6–10): Skills Plan + SQL set + dashboard prototype
  • Release 3 (Week 11–14): Python automation + ITSM SOP + Career Package

Approach:

  • Predictive: Milestone dates and required artifacts.
  • Iterative: Multiple feedback cycles for analytics and documentation artifacts.
  • Incremental: Deliver usable outputs at each release.

🛠 Tool Selection & Setup

Tools Explored

  • Microsoft Project → Predictive planning and WBS reporting; heavy for solo student use.
  • Trello → Lightweight Kanban; limited reporting and dependency handling.
  • Jira → Best mix for Hybrid delivery (backlog, sprints, releases, and reporting).

Final Choice: Jira

Jira supports the Hybrid plan: fixed milestones with iterative artifacts and continuous feedback.

  • Backlog & Sprints → manage Skill Plan tasks and artifacts.
  • Releases → R1, R2, R3 cadence aligned to this charter.
  • Custom issue types → SQL, Dashboard, Python, SOP, Resume, LinkedIn.

Dependencies

  • Role research → precedes GAP analysis and Skill Plan.
  • SQL practice → precedes dashboard and Python automation.
  • Artifacts → precede resume/LinkedIn updates.

Example Backlog

  • Extract competencies from three postings
  • Complete GAP matrix and prioritize
  • Build SQL practice set
  • Create dashboard prototype and iterate
  • Write ITSM SOP for ticket/asset workflow
  • Automate CSV-to-dashboard update with Python
  • Update resume + LinkedIn and draft cover letters
  • Integrate artifacts into portfolio
