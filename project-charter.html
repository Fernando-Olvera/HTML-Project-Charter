    üß≠ Project Methodology

    Requirements: Role analysis and deliverables are well-defined, but artifacts (dashboard, SQL set, Python automation) benefit from iteration and feedback.

    Delivery Cadence: Three releases to reduce risk and incorporate feedback.

    • Release 1 (Week 3‚Äì5): Role research matrix + GAP analysis draft
    • Release 2 (Week 6‚Äì10): Skills Plan + SQL Cert + Python Cert
    • Release 3 (Week 11‚Äì14): IT Fundamentals Cert + Career Package + Application Submissions

    Approach:

    • Predictive: Milestone dates and required artifacts.
    • Iterative: Multiple feedback cycles for analytics and documentation artifacts.
    • Incremental: Deliver usable outputs at each release.

    Tools Explored

    • Microsoft Project ‚Üí Predictive planning and WBS reporting; heavy for solo student use.
    • Trello ‚Üí Lightweight Kanban; limited reporting and dependency handling.
    • Jira ‚Üí Best mix for Hybrid delivery (backlog, sprints, releases, and reporting).

    • Backlog & Sprints ‚Üí manage Skill Plan tasks and artifacts.
    • Releases ‚Üí R1, R2, R3 cadence aligned to this charter.
    • Custom issue types ‚Üí SQL, Dashboard, Python, SOP, Resume, LinkedIn.

    Dependencies

    • SQL practice ‚Üí precedes dashboard and Python automation.
